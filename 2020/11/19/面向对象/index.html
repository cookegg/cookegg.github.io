面向对象

面向对象简介

对象是客观世界中的物体在人脑中的映像，和人为的概念。

所谓（软件）对象，是一种将状态（数据）和行为（操作）合成到一起的软件构造，用来描述真实世界的一个（物理或概念性的）对象。

对象操作也可看作是可以向对象要求提供的服务（c#编程语言称作方法）

类是对众多对象抽象出来的一个模型 ，把模型实例化后就是一个对象

定义类及类成员

静态成员属于类，实例成员属于对象

public公有访问，不受任何限制

private私有访问。只限于本类成员内访问，子类，实例都不能访问。

static静态化，只可在类中使用不可进行实例化（不常用）

protected保护访问。只限于本类内部和子类内部可访问，实例不能访问

internal内部访问。只限于本地项目内访问，其他不能访问。（相当于项目内的public）

protected internal内部保护访问。只限于本地项目和子类访问，其他不能访问。（protected 和 internal权限之和）

继承

  1. 被继承的类成为父类，基类，超类，而主动继承的类成为子类或派生类。子类继承父类的状态和行为，同时也有自己的特性

  2. System.Object是所有类型的基类

  3. C#中继承的写法，class A:B{}

  4. 继承具有传递性，由A:B B:C => A:C

  5. 构造方法不能继承

  6. C#只允许单继承，一个类只能继承与一个父类

  7. 密封函数不能被继承，例如：sealed class Animal{}

  8. 子类不仅继承了父类的共有成员，同时继承了父类的私有成员，只是父类的私有成员在子类中不可被访问

多态

重载

同一个类中有多个方法的名称相同，参数个数不同或类型不同，则属于重载。

同一个类中有多个方法的名称相同，参数个数相同，参数类型相同，返回值类型不同，这样不属于重载，编译时会报错。

重写（override）

普通类中的virtual方法可在子类中被重写；

抽象类重点abstract方法和abstract属性必须直接或间接在子类中被重写；

接口中的所有成员都必须被子类实现；
